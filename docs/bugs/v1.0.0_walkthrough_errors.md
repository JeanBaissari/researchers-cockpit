# Known errors hapenning as of 12/23/2025:

## 1 ✅ FIXED - Syntax Error in lib/plots.py

**Status**: ✅ RESOLVED (2025-01-23)

**Original Error**: When running commands such as `python3 scripts/ingest_data.py --source yahoo --symbols SPY --assets equities` or `python3 -c "from lib.data_loader import list_bundles; print(list_bundles())"`, we received:

```
SyntaxError: parameter without a default follows parameter with a default
```

**Root Cause**: The `plot_all()` function in `lib/plots.py` had a required parameter (`save_dir`) positioned after optional parameters, violating Python syntax rules.

**Solution**: Moved `save_dir` parameter before optional parameters in the function signature.

**Files Modified**:
- `lib/plots.py` - Fixed function signature (line 289-295)
- `lib/backtest.py` - Updated function call to match new parameter order (line 349-355)

**Verification**: 
```bash
source venv/bin/activate
python3 -c "from lib.plots import plot_all; print('✓ Import successful')"
```

**Documentation**: See `docs/troubleshooting/syntax_error_plots_import.md` for details.

## 2 ✅ FIXED - Sharpe Ratio API Parameter Error

**Status**: ✅ RESOLVED (2025-01-23)

**Original Error**: When running backtests, received:
```
TypeError: sharpe_ratio() got an unexpected keyword argument 'risk_free_rate'
```

**Root Cause**: The `empyrical` library uses different parameter names:
- `sharpe_ratio()` uses `risk_free` (not `risk_free_rate`)
- `sortino_ratio()` uses `required_return` (not `risk_free_rate`)
- `alpha()`, `beta()`, `omega_ratio()` use `risk_free` (not `risk_free_rate`)

**Solution**: Updated all empyrical function calls in `lib/metrics.py` to use correct parameter names.

**Files Modified**:
- `lib/metrics.py` - Fixed all empyrical API calls (lines 66, 76, 109, 110, 116)

**Verification**: 
```bash
python scripts/run_backtest.py --strategy spy_sma_cross --start 2020-01-01 --end 2025-12-01
```

**Documentation**: See `docs/troubleshooting/sharpe_ratio_api_error.md` for details.
